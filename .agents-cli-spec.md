# Agent Spec: Omni-Agent (`omni-agent`)

This specification is aligned with the authoritative project specifications [SPEC.md](file:///usr/local/google/home/nicolasalvo/projects/dev/ge_agents/omni-agent/SPEC.md) and [TEST_SPEC.md](file:///usr/local/google/home/nicolasalvo/projects/dev/ge_agents/omni-agent/TEST_SPEC.md).

## Overview
The **Omni-Agent** is an intelligent, intent-driven video creation and editing assistant built for **Gemini Enterprise App** users. It enables users to generate new videos from text or images and iteratively refine or edit existing videos through natural conversation.

### Architectural Separation of Concerns
* **Main Orchestration Agent:** Runs a fast reasoning model (`AGENT_MODEL_ID`, e.g., Gemini Flash 3.5 / Gemini 2.5 Flash). It handles user intent classification, conversational context, interactive prompt rewriting for video generation assets, and session state orchestration (`previous_interaction_id`, `file_data_mappings`).
* **Video Generation & Editing Engine:** Powered exclusively by **Gemini Omni (`gemini-omni-flash-preview` / `OMNI_MODEL_ID`)** via the Google GenAI Interactions API (`client.interactions.create(...)`). The main agent invokes Gemini Omni as a dedicated **Tool** (`video_generation_tool`).

## Example Use Cases & Critical User Journeys (CUJs)
1. **CUJ-1 (Text-to-Video Generation):** User inputs detailed prompt (`16:9` / `9:16`). Main Agent invokes `video_generation_tool(task="text_to_video")`, stores `interaction.id` in the ADK Session Service, writes `.mp4` to GCS, and returns inline `gs://` player + HTTPS download link.
2. **CUJ-2 (Image-to-Video Animation):** User uploads local image (`tests/fixtures/sample_image.png`) with motion prompt. `FileDataResolverPlugin` resolves GCS URI; Main Agent invokes `video_generation_tool(task="image_to_video", file_uris=[...])`, stores `interaction.id` in ADK Session Service, and returns dual-link output.
3. **CUJ-3 (Iterative Stateful Edit):** User requests follow-up edit (*"Make the lighting sunset golden hour"*). Main Agent retrieves stored `previous_interaction_id` from the ADK Session Service and passes it to `video_generation_tool(task="edit")` across turns.
4. **CUJ-4 (Uploaded Video Edit):** User uploads source `.mp4` (`tests/fixtures/sample_video.mp4`) with edit instructions. Main Agent resolves uploaded file URI, invokes `video_generation_tool(task="edit", file_uris=[...])`, and stores `interaction.id` in ADK Session Service.
5. **CUJ-5 (Asset Prompt Rewrite Gate):** User enters underspecified prompt (*"make a car video"*). Main Agent intercepts execution, presents enriched rewrite + 3-way HITL choice (*1. Use Re-written / 2. Use Original / 3. Amend*).
6. **CUJ-6 (Safety Block Handling):** Prompt triggers `FINISH_REASON_SAFETY`. Tool captures API exception cleanly and returns exact safety feedback without retrying or crashing.

## Tools Required
* **`video_generation_tool`**:
  * Explicit typed parameter signature:
    ```python
    def video_generation_tool(
        prompt: str,
        task: str,  # "text_to_video", "image_to_video", "reference_to_video", or "edit"
        aspect_ratio: str = "16:9",
        file_uris: list[str] | None = None,
        previous_interaction_id: str | None = None,
        tool_context: ToolContext = None,
    ) -> str:
    ```
  * Calls `client.interactions.create(...)` targeting `OMNI_MODEL_ID`.
  * Explicitly stores the returned `interaction.id` (`Interaction ID`) in the **ADK Session Service (`tool_context.session.state["previous_interaction_id"] = interaction.id`)**.
  * Saves `.mp4` artifacts to `gs://<GCS_BUCKET_NAME>/artifacts/<uuid>.mp4`.
  * Returns dual-link format: Inline markdown player (`![Video](gs://...)`) and authenticated HTTPS download URL (`https://storage.cloud.google.com/...`).

## Plugins & State Required
* **`FileDataResolverPlugin`**: ADK custom plugin intercepting uploaded `FileData` / `inline_data` parts and recording `filename -> gs://...` mappings in `session.state["file_data_mappings"]`.
* **ADK Session Service Interaction ID Persistence**:
  * Every time `video_generation_tool` executes `interaction = client.interactions.create(...)`, it explicitly records the returned Interaction ID in the ADK Session Service: `tool_context.session.state["previous_interaction_id"] = interaction.id`.
  * Multi-turn stateful edits (`edit`) retrieve `tool_context.session.state["previous_interaction_id"]` and pass it to `client.interactions.create(..., previous_interaction_id=previous_interaction_id)` across turns (`CUJ-3` and `CUJ-4`).
  * Users can reset state (`tool_context.session.state["previous_interaction_id"] = None`) by requesting *"New video"* or *"Start over"*.

## Environment & Configuration (Zero Hardcoding)
All runtime settings must be externalized via `.env`:
* `GOOGLE_CLOUD_PROJECT`
* `GOOGLE_CLOUD_REGION` (`global`)
* `AGENT_MODEL_ID` (`gemini-2.5-flash` or configured Flash 3.5 string)
* `OMNI_MODEL_ID` (`gemini-omni-flash-preview`)
* `GCS_BUCKET_NAME`

## Constraints & Safety Rules
* Report safety blocks (`FINISH_REASON_SAFETY`) cleanly to the user without retrying or hallucinating output.
* Do not inject artificial corporate watermarks or unrequested style modifiers.
* Strictly enforce Zero Hardcoding.

## Success Criteria & Definition of Done
1. **Unit Testing:** 100% passing unit tests for tool logic, prompt rewriting gate, `FileDataResolverPlugin`, ADK Session Service state persistence, and safety rejection handling.
2. **End-to-End Testing (`TEST_SPEC.md`):** Complete passing verification of all 6 Critical User Journeys (CUJ-1 through CUJ-6) using `agents-cli run --session-id <id>` and `--file <path>` local file upload testing with local fixtures (`tests/fixtures/sample_image.png` and `sample_video.mp4`).
