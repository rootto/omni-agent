# Omni-Agent

Omni-Agent is an AI-powered conversational video generation and editing assistant built on top of **Google's Gemini Omni**, an advanced natively multimodal AI model. 

*Gemini Omni* goes beyond simple text-to-video mapping by functioning as a unified system capable of processing and generating text, images, audio, and video simultaneously. Equipped with omni-modal reasoning, this agent can execute complex, conversational, step-by-step video editing operations. Whether it's swapping a background, adjusting lighting, maintaining scene consistency across multiple chat turns, or using image payloads to inform cinematic control, Omni-Agent exposes these capabilities seamlessly.

This agent relies on **Gemini Omni Flash preview** to execute high-speed cinematic generation and video logic through the Agent Runtime in Vertex AI.

## Quick Start

Install `agents-cli` and its skills if not already installed:

```bash
uvx google-agents-cli setup
```

Copy the `.env.example` file to create your own local `.env` configuration file, and populate the placeholder fields with your actual details. 
The necessary variables include:
- `GOOGLE_CLOUD_PROJECT`: Your Google Cloud Project ID (e.g. `my-awesome-project`)
- `GOOGLE_CLOUD_PROJECT_NUMBER`: Your numeric Google Cloud Project Number (e.g. `123456789`)
- `AGENT_RUNTIME_SERVICE_ACCOUNT`: (Optional) The service account Vertex AI Reasoning Engine runs as (defaults to `service-<PROJECT_NUMBER>@gcp-sa-aiplatform-re.iam.gserviceaccount.com`).
- `COMPUTE_SERVICE_ACCOUNT`: (Optional) The default Compute Engine service account used as a fallback (defaults to `<PROJECT_NUMBER>-compute@developer.gserviceaccount.com`).
- `GEMINI_ENTERPRISE_INSTANCE`: The name of your initialized Gemini Enterprise instance
- `GEMINI_ENTERPRISE_APP_ID`: Your target Agent App ID (can be autogenerated by the deployment script if `GOOGLE_CLOUD_PROJECT_NUMBER` and `GEMINI_ENTERPRISE_INSTANCE` are present)
- `GCS_BUCKET_NAME` & `LOGS_BUCKET_NAME`: The Google Cloud Storage buckets for video artifacts and logs (e.g. `my_geapp_bucket`)

```bash
cp .env.example .env
```

Install required packages:

```bash
agents-cli install
```

Test the agent with a local web server:

```bash
agents-cli playground
```

You can also use features from the [ADK](https://adk.dev/) CLI with `uv run adk`.

## Development

Edit your agent logic in `app/agent.py` and test with `agents-cli playground` - it auto-reloads on save.

## Deployment

The `deploy-agent.sh` script automates the deployment process and will automatically enable the required `cloudresourcemanager.googleapis.com` API for telemetry before launching the Reasoning Engines backend.

To deploy manually using the CLI:
```bash
gcloud config set project <your-project-id>
gcloud services enable cloudresourcemanager.googleapis.com
agents-cli deploy
```

To add CI/CD and Terraform, run `agents-cli scaffold enhance`.
To set up your production infrastructure, run `agents-cli infra cicd`.
